現在いるブランチの変更内容（ベースブランチとの差分）を分析し、GitHubのPull Requestを作成する。**PRのタイトルは、ユーザーが自分で入力するか、Claudeが提案したものを承認する形で決定する。** 本文(説明)は変更内容から自動で組み立てて提示し、承認を得てから作成する。

## 前提

- `gh` CLI が利用可能で、認証済みであること。
- 現在のブランチがベースブランチと異なり、差分（コミット）があること。

## 手順

### 1. ブランチと差分を確認する（許可不要）

```
git branch --show-current
gh repo view --json defaultBranchRef -q .defaultBranchRef.name
```
- 現在のブランチとベースブランチ（デフォルトブランチ）を把握する。
- ベースとの差分を確認する:
```
git log --oneline <base>..HEAD
git diff <base>...HEAD --stat
```
- 差分が無い場合は、その旨を伝えて終了する。
- 既に同じブランチのPRが存在する場合は、その旨を伝えて終了する（`gh pr view` で確認）。

### 2. 変更内容を分析する（許可不要）

- コミット履歴と差分から、このブランチが何を変更したか（目的・主な変更点）を把握する。

### 3. PRタイトルを決定する　→【タイトル決定ポイント】

**タイトルは必ずユーザーの意思で決定する。** 次の手順で進める:

1. 変更内容に基づいて、**タイトル案を1つ（必要なら複数）提案**する。
2. ユーザーに、以下のどちらかを選んでもらう:
   - **提案を承認する**（提案タイトルをそのまま採用）。
   - **自分でタイトルを入力する**（ユーザーが入力した文字列を採用）。
3. ユーザーがタイトルを確定するまで、PRは作成しない。

> タイトルはユーザーが入力 or 提案を承認、のいずれかで必ず確定させること。Claudeが勝手にタイトルを決めて作成しない。

### 4. PR本文を組み立てて提示する

変更内容から本文(説明)を組み立てる。例:
- **概要**: このPRで何をするか。
- **変更点**: 主な変更を箇条書き。
- **備考 / 確認事項**（あれば）。

確定したタイトルと、組み立てた本文をユーザーに提示する。

### 5. PRを作成する（タイトル確定・本文承認後）

未pushのコミットがあれば先にpushする（必要に応じて確認）:
```
git push -u origin <current-branch>
```
PRを作成する:
```
gh pr create --base <base> --head <current-branch> --title "<確定したタイトル>" --body "<本文>"
```
- 作成後、PRのURLを報告する。

## 注意

- 説明・本文・報告はすべて日本語で行う（タイトルはユーザー指定があればそれに従う）。
- **タイトルは必ず「ユーザー入力」または「提案の承認」で決定する。** 勝手に決めない。
- 本文は提案だが、作成前にユーザーへ提示して承認を得る。
- `git push` や `gh pr create` は外部への反映を伴うため、ユーザーの確定・承認後に実行する。
- draftにしたい場合は `--draft` を付ける（ユーザーの希望があれば）。

---

## 使い方

Claude Code上で以下を入力するだけで実行されます。
/create-pr

最初からタイトルを指定することもできます。
例: 「/create-pr タイトルは『通知機能の追加』で」
