Gitにステージングされている変更（`git add` 済みのもの）を分析し、適切なコミットメッセージを日本語で考えて提案する。

## 手順

1. ステージングされている変更を確認する:
   ```
   git diff --cached
   ```
   - 何もステージングされていない場合は、その旨をユーザーに伝えて終了する（`git add` を促す）。

2. 変更内容を分析する:
   - 変更されたファイルと差分の内容を読み取る。
   - 何を・なぜ変更したのかを把握する（機能追加・バグ修正・リファクタリング・ドキュメント更新など）。

3. コミットメッセージを Conventional Commits 形式で組み立てる:
   - **1行目（subject）**: `<type>: <要約>` の形式。日本語で簡潔に（50文字程度まで）。
     - type の例: `feat`（機能追加）, `fix`（バグ修正）, `docs`（ドキュメント）, `refactor`（リファクタリング）, `style`（整形）, `test`（テスト）, `chore`（雑務・設定）
   - 必要に応じて空行を挟み、**本文（body）** で変更の理由や詳細を箇条書きで説明する。

4. 提案したコミットメッセージをユーザーに提示する。
   - そのままコミットして良いか確認する。
   - 承認されたら以下を実行する:
     ```
     git commit -m "<subject>" -m "<body>"
     ```
   - 承認されるまで `git commit` は実行しない。

## 注意

- コミットメッセージは必ず日本語で作成する。
- 変更が複数の異なる目的を含む場合は、その旨を指摘し、分割を提案する。
- ユーザーから修正の指示があれば、メッセージを調整して再提案する。
