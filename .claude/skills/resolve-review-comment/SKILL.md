現在いるブランチに紐づくGitHub PRから、**まだ修正していないレビューコメントを1件**収集し、その妥当性を判断したうえで、修正案の提示 → 修正 → ステージ → コミット → push → 該当レビューコメントへの返信、までを一連で行う。

## 前提

- `gh` CLI が利用可能で、認証済みであること。
- 現在のブランチにPRが存在すること。

## 許可ポリシー（重要）

このSkillは、**以下の2つのタイミングでのみユーザーに許可を求める。それ以外は許可を求めず処理を続行する。**

1. **修正案を提示した後**、ファイルを編集する前（編集してよいかの許可）。
2. **修正した後**、ステージ・コミット・push・返信を行う前（コミット/push/返信してよいかの許可）。

上記2点以外（PRの特定、コメント収集、妥当性判断、各コマンドの実行など）では、いちいち確認せずに進める。

## 手順

### 1. 対象PRを特定する（許可不要）

```
gh pr view --json number,title,headRefName,baseRefName,url
```
- PRが見つからない場合は、その旨を伝えて終了する。
- `owner`/`repo` は `gh repo view --json owner,name` で取得する。

### 2. 未対応のレビューコメントを1件収集する（許可不要）

PRのレビューコメント（差分上のインラインコメント）を取得する:
```
gh api repos/{owner}/{repo}/pulls/{number}/comments
```
- 各コメントの `id`, `path`, `line`（または `original_line`）, `body`, `user.login`, `in_reply_to_id` を確認する。
- 「まだ修正していないコメント」を1件選ぶ。判定基準:
  - そのコメントに対する自分（bot/ユーザー）からの返信スレッドが無い、または `fix(...)` 返信が付いていないもの。
  - すでに `fix(<hash>)` の返信が付いているコメントは対応済みとみなしスキップする。
  - 返信コメント（`in_reply_to_id` があるもの）自体は対象にしない。
- 未対応コメントが無い場合は、その旨を伝えて終了する。
- 複数ある場合は、最も古い未対応コメント1件を対象にする（どれを対象にしたか明示する）。

### 3. コメントの妥当性を判断する（許可不要）

- 該当 `path`/`line` のコードを読み、コメントの指摘が妥当か判断する。
- 妥当な場合: どう修正するかの方針を決める。
- 妥当でない（修正すべきでない）と判断した場合: その理由を説明し、修正は行わず、対応をユーザーに委ねて終了する（勝手に却下返信はしない）。

### 4. 修正案を提示する　→【許可ポイント①】

- 対象コメントの内容、妥当性の判断、具体的な修正案（変更前後のコード）を日本語で提示する。
- **ここでファイル編集の許可を求める。** 承認されるまで編集しない。

### 5. 修正する（許可不要で実行）

- 承認後、提示した修正案どおりにファイルを編集する。
- 何を変更したかを日本語で説明する。

### 6. 修正後　→【許可ポイント②】

- 修正内容を提示し、**ここで「ステージ・コミット・push・返信を実行してよいか」の許可を求める。**
- 承認されるまで以降のgit操作・返信は行わない。

### 7. ステージ・コミット・push（許可不要で実行）

承認後、以下を実行する:
```
git add <修正したファイル>
git commit -m "fix: <レビュー指摘に対応した修正内容（日本語）>"
git push
```
- push後、生成されたコミットハッシュを取得する:
```
git rev-parse HEAD
```

### 8. レビューコメントに返信する（許可不要で実行）

取得したコミットハッシュを使い、対象レビューコメントに返信する:
```
gh api repos/{owner}/{repo}/pulls/{number}/comments/{comment_id}/replies \
  -f body="fix(<コミットハッシュ>)"
```
- 返信本文は `fix(<コミットハッシュ>)` の形式（例: `fix(975064d)`）。
- 返信完了後、対応したコメント・コミット・返信内容を日本語で報告する。

## 注意

- 説明・コミットメッセージ・報告はすべて日本語で行う。
- 許可を求めるのは【許可ポイント①】【許可ポイント②】の2箇所のみ。それ以外では確認せず進める。
- 1回の実行で対応するレビューコメントは1件のみ。
- コミットメッセージは指摘内容を反映した `fix:` を基本とする。
- push先ブランチが未設定（upstream無し）の場合は `git push -u origin <現在のブランチ>` を使う。
