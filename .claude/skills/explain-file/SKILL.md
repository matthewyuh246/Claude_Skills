ユーザーが指定したファイルを読み込み、**1行ずつ、できるだけわかりやすく**日本語で説明する。プログラミング初心者にも理解できるよう、専門用語には噛み砕いた補足を添える。

## 前提

- ユーザーが説明対象のファイルを指定していること。
  - 指定が無い場合は、どのファイルを説明すればよいかユーザーに尋ねる。
  - IDEで開いているファイルがある場合は、それを対象候補として確認する。

## 手順

### 1. ファイルを読み込む

- 指定されたファイルを読み込み、全体の行数を把握する。
- ファイルが存在しない場合は、その旨を伝えて終了する。
- 行数が非常に多い場合（目安200行超）は、一度に全部やると読みにくいため、範囲を区切って説明することを提案する（例: 「1〜50行ずつ進めますか?」）。

### 2. 全体像を先に伝える

1行ずつの説明に入る前に、まず短く全体像を示す:
- このファイルが何をするものか（役割・目的）。
- 使われている言語・フレームワーク。
- 大まかな構成（どんなブロックに分かれているか）。

### 3. 1行ずつ説明する

各行について、以下の形式で説明する:

```
行番号 | コード
→ 説明（その行が何をしているか、なぜ必要か）
```

説明の方針:
- **1行ごと**に、その行が何をしているかを日本語で説明する。
- 専門用語（変数、関数、ループ、インポート等）が出たら、初出時に簡単な意味を添える。
- 単純な行（閉じ括弧だけ、空行など）はまとめて軽く触れてよいが、意味のある行は省略しない。
- 複数行で1つの意味を成す場合（関数定義のブロックなど）は、まずブロック全体の役割を述べてから各行を説明する。
- コードだけでなく、設定ファイル・マークダウン・データファイルなども対象にする。

### 4. 補足・まとめ

- 全体を通して、初心者がつまずきやすい点や、特に重要な行があれば最後に補足する。
- 必要に応じて、関連する概念へのごく簡単な説明を添える。

## 注意

- 説明はすべて日本語で行う。
- 「できるだけわかりやすく」が最優先。専門的に正確であっても難解な表現は避け、噛み砕く。
- 推測で断定しない。意図が不明な行は「おそらく〜」と前置きし、断定を避ける。
- ファイルの内容を編集・変更はしない（説明のみ）。

---

## 使い方

Claude Code上で以下のように入力します。
/explain-file

対象ファイルを添えて指定することもできます。
例: 「/explain-file src/main.py を説明して」
