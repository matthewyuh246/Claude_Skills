現在いるブランチに紐づくGitHub PRの変更ファイルをレビューし、**確実に修正した方がよい箇所のみ**を最小限に絞って指摘し、GitHubのPRにコメントを投稿する。

## 前提

- `gh` CLI が利用可能で、認証済みであること（`gh auth status` で確認できる）。
- 現在のブランチにPRが存在すること。

## レビュー方針（重要）

- **最小限の指摘に徹する。** 数を出すことが目的ではない。
- 指摘するのは「**確実に修正した方がよい**」もののみに限定する:
  - バグ・ロジックの誤り（明確に動作がおかしくなるもの）
  - Nullや例外で落ちる箇所、境界条件の漏れ
  - セキュリティ上の問題（機密情報の混入、インジェクション等）
  - リソースリーク、明らかな性能上の致命的問題
  - 明確な誤り（typo含む）で動作・意味に影響するもの
- **指摘しない**もの:
  - 好みレベルのスタイル・命名（動作に影響しないもの）
  - 「こうもできる」程度の代替案
  - 確信が持てない推測ベースの指摘
- 確実に修正すべき箇所が無ければ、無理に指摘を作らず **`LGTM` とコメント**する（自分のPRなので approve はしない）。

## 手順

### 1. 対象PRを特定する

現在のブランチのPRを確認する:
```
gh pr view --json number,title,headRefName,baseRefName
```
- PRが見つからない場合は、その旨を伝えて終了する。

### 2. 変更ファイルと差分を取得する

```
gh pr diff
```
- 変更されたファイル一覧と差分を読み取る。
- 文脈が必要な場合は、該当ファイルをローカルで開いて前後を確認する。

### 3. 差分をレビューする

- 上記「レビュー方針」に従い、確実に修正すべき箇所のみを抽出する。
- 各指摘について、ファイル名・行・問題内容・修正案を整理する。
- 指摘は必ず差分（追加・変更された行）に基づいて行う。

### 4. 指摘内容をまとめる

投稿するコメント内容を日本語でまとめる:
- 指摘箇所ごとに: **ファイル:行 / 問題 / なぜ直すべきか / 修正案**
- 指摘が無い場合は **`LGTM`**（確実に修正すべき箇所は無い旨）とする。
- まとめた内容はユーザーへの報告として提示してよいが、**投稿前の承認確認は不要**。そのまま手順5で投稿する。

### 5. GitHubのPRにコメントする（承認不要・そのまま実行）

状況に応じて以下のいずれかで投稿する。**いずれの場合も、コメント本文の末尾に必ず次の署名を付ける**（Claudeがコメントしたと分かるようにするため）:

```

---
🤖 Reviewed by Claude Code
```

- **確実に修正すべき箇所が無い場合（LGTM）**: `LGTM` とコメントする。
  ```
  gh pr review --comment -b "LGTM

---
🤖 Reviewed by Claude Code"
  ```
  - **自分で作成したPRを想定しているため approve（`--approve`）はしない。** 必ず `--comment` で投稿する。

- **特定の行へのインラインコメント**（指摘がある場合に推奨。複数指摘をまとめて1レビューにする）:
  `gh api` で Pull Request review を作成する。
  ```
  gh api repos/{owner}/{repo}/pulls/{pr-number}/reviews \
    -f event=COMMENT \
    -f body="<レビュー全体の概要>

---
🤖 Reviewed by Claude Code" \
    -f 'comments[][path]=<ファイルパス>' \
    -f 'comments[][line]=<行番号>' \
    -f 'comments[][side]=RIGHT' \
    -f 'comments[][body]=<指摘内容と修正案>'
  ```
  - `{owner}/{repo}` は `gh repo view --json owner,name` 等で取得する。
  - 行番号は差分の新しい側（RIGHT）の行を指定する。
  - 複数指摘がある場合は `comments[]` を繰り返して1回のレビューにまとめる。
  - レビュー全体の概要（`body`）の末尾に署名を付ける。

- 行指定が難しい・全体への総評のみの場合は、本文コメントを使う（末尾に署名を付ける）:
  ```
  gh pr review --comment -b "<コメント本文>

---
🤖 Reviewed by Claude Code"
  ```

- 投稿後、コメントしたPRのURLを報告する。

## 注意

- 指摘・コメントはすべて日本語で行う。
- 推測で指摘しない。確信が持てるもののみコメントする。
- コメントは常に `--comment`（レビュー意見）で行う。**このSkillは自分で作成したPRを想定しているため、`--approve` は使わない**（LGTMの場合も `--comment -b "LGTM"` で投稿する）。`--request-changes` も使わない。
- **すべてのコメント本文の末尾に `🤖 Reviewed by Claude Code` の署名を付ける**（Claudeがコメントしたと分かるようにするため）。
- **投稿前のユーザー承認は不要。** レビューがまとまったらそのまま投稿してよい。
