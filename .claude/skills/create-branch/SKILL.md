自分にアサインされているGitHub issueを一覧で取得し、番号を振ってどのissueを対象にするかユーザーに選んでもらう。選ばれたissueに対して、**現在いるブランチを起点**に、命名ルールに則った作業ブランチを発行（作成してチェックアウト）する。

## ブランチ命名ルール

```
feature/<issue番号>-<短い説明>
fix/<issue番号>-<短い説明>
hotfix/<issue番号>-<短い説明>
chore/<issue番号>-<短い説明>
refactor/<issue番号>-<短い説明>
```

- **プレフィックス**: 上記5種類（`feature` / `fix` / `hotfix` / `chore` / `refactor`）から選ぶ。
- **issue番号**: 対象issueの番号（`#` は付けず数字のみ）。
- **短い説明**: issueタイトルを元にした英小文字のkebab-case。スペースは `-` に、記号は除去する。簡潔に（おおむね3〜5語まで）。

例: issue #42「ログイン画面のバリデーション修正」→ `fix/42-login-validation`

## 前提

- `gh` CLI が利用可能で、認証済みであること。
- GitHubリポジトリ内であること。
- 作業ツリーがクリーンであること（未コミットの変更があれば注意喚起する）。

## 手順

### 1. アサインされているissueを取得する（許可不要）

自分にアサインされているオープンなissueをすべて取得する:
```
gh issue list --assignee "@me" --state open --json number,title,labels,updatedAt
```
- 1件も無い場合は、その旨を伝えて終了する。
- ユーザーが最初から特定のissue番号を指定している場合は、この一覧提示・選択を省略してそのissueを対象にしてよい（`gh issue view <number> --json number,title` で存在確認する）。

### 2. 対象issueをユーザーに選んでもらう

- 取得したissueに **1, 2, 3 … と通し番号を振り**、番号・issue番号・タイトルを一覧で提示する。
  - 例: `1. #42 ログイン画面のバリデーション修正`
- **どのissueを対象にするかユーザーに尋ねる。** ユーザーが選ぶまで先に進まない。
- ユーザーは通し番号でも issue番号（#付き/なし）でも指定できるものとする。

### 3. 起点ブランチと作業ツリーを確認する（許可不要）

```
git branch --show-current
git status --porcelain
```
- 現在のブランチが起点になる旨をユーザーに伝える。
- 未コミットの変更がある場合は、そのまま起点ブランチから分岐する点に注意喚起する（必要ならユーザーに続行可否を確認する）。

### 4. プレフィックスと短い説明を決定する　→【ブランチ名決定ポイント】

1. issueタイトルとラベルから、**適切なプレフィックスを提案**する（labels に `bug` があれば `fix` を提案、など）。
2. issueタイトルを元に **短い説明（英小文字kebab-case）を生成**する。日本語タイトルの場合は意味を汲んで英語の短い説明に変換する。
3. 上記から組み立てた **ブランチ名の案を提示**し、ユーザーに以下のどちらかを選んでもらう:
   - **提案を承認する**（提案ブランチ名をそのまま採用）。
   - **修正を指示する**（プレフィックス・短い説明・全体を上書き）。
4. ユーザーがブランチ名を確定するまで、ブランチは作成しない。

> ブランチ名は必ず「ユーザー入力」または「提案の承認」で確定させること。勝手に作成しない。

### 5. ブランチを発行する（ブランチ名確定後）

確定したブランチ名で、**現在のブランチを起点**に新しいブランチを作成してチェックアウトする:
```
git switch -c <確定したブランチ名>
```
- 既に同名のブランチが存在する場合は作成せず、その旨を伝えて別名を促す。
- 作成後、新しいブランチ名・起点ブランチ・対象issue番号を報告する。

## 注意

- 説明・提案・報告はすべて日本語で行う（ブランチ名は命名ルールに従い英数字）。
- **ブランチ名は必ず「ユーザー入力」または「提案の承認」で決定する。** 勝手に決めない。
- 起点は常に **現在いるブランチ**。デフォルトブランチへ勝手に切り替えたりはしない（ユーザーが希望した場合のみ）。
- `git switch -c` 以外の破壊的操作（stash・reset・force等）は行わない。
- リモートへのpushはこのSkillでは行わない（必要ならユーザーの指示で別途）。

---

## 使い方

Claude Code上で以下を入力するだけで実行されます。
/create-branch

最初から対象issueやプレフィックスを指定することもできます。
例: 「/create-branch #42 を fix ブランチで」
